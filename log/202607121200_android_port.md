# Android port: examples app (M1 cross-compile + M2 refactor + app)

## Goal

Run the six examples on Android (one app, example picker + live view),
initial target devices: Snapdragon 8 Gen 2 (Adreno 740) and Dimensity 9500
(Arm G1-Ultra). Plan: 1アプリ統合 / ANativeWindow_lock software blit (no
EGL, no second Vulkan context) / Kotlin + JNI / arm64-v8a, minSdk 29.

## What was done

### M1 — NDK cross-compile of the engine (no Gradle)

- Installed cmdline-tools + `ndk;28.2.13676358` + `cmake;3.31.6` into the
  existing Android SDK (`%LOCALAPPDATA%/Android/Sdk`).
- Root `CMakeLists.txt` Android branches (desktop paths byte-identical):
  - `FindVulkanSdk` + `find_package(Vulkan)` skipped on Android — the
    fetched `Vulkan::Headers` would collide with FindVulkan's alias, and
    with `VULKAN_HPP_DISPATCH_LOADER_DYNAMIC` only the NDK's `libvulkan.so`
    (linked as plain `vulkan`) is needed.
  - glslang/SPIRV-Tools/SPIRV-Headers/**Vulkan-Headers** FetchContent-built
    from the Khronos repos at the lockstep tag `vulkan-sdk-1.4.350.0`
    (matches the desktop SDK 1.4.350.0). The NDK sysroot has no arm64
    glslang and only the C Vulkan headers (`vulkan.hpp` absent — the
    fetched header set shadows the sysroot's `vulkan.h` so .h/.hpp stay
    consistent). `SPIRV-Headers_SOURCE_DIR` must be set manually (CMake
    FetchContent only defines the lowercased variable).
  - GLFW fetch + `dressi_glad` wrapped in `if(NOT ANDROID)`; `dressi_vkw`
    links `Vulkan::Headers vulkan android log` on Android instead of
    `Vulkan::Vulkan glfw`.
  - `FetchAssets` additionally gated on `NOT ANDROID`.
- Verified: `libdressi.a` / `libdressi_vkw.a` build clean for
  arm64 (`elf64-littleaarch64`), Ninja + NDK r28.2 toolchain, C++20.
- Desktop regression: `cmake --preset msvc` + full build + `ctest --preset
  release` all green (89 cpu + 43 gpu).

### M2a — examples refactor (desktop behavior unchanged)

- New abstraction in `examples/common/`: `viewer.h` (`IViewer`:
  valid/update/setTitle/setPosition), `example_host.h` (`ExampleHost`:
  `makeViewer` + `cancelled`, `RunFn`, `ExampleInfo`,
  `GetExampleRegistry()`), `example_registry.cpp`, `desktop_host.{h,cpp}`
  (wraps the untouched `VkViewer`).
- Each `examples/<name>/main.cpp` → `run.cpp` (`git mv`, then surgical
  edits): body became `Run<Name>(const std::vector<std::string>&,
  ExampleHost&)`, viewers go through `host.makeViewer` (`.` → `->`), one
  `if (host.cancelled()) break;` per optimization/frame loop, `printf` →
  spdlog (logcat). Thin `main.cpp` (argv → DesktopHost) per example.
- Arg normalization (defaults preserved): `--data-dir=`/`--out-dir=`
  everywhere assets/outputs are used; `texture_optimization` gained
  `--views=`/`--iters=` (was hardcoded 6/4000; positional data dir kept);
  `pbr_material_optimization` gained `--views=`.
- `examples/CMakeLists.txt`: portable `dressi_examples_core` STATIC (all
  run.cpps + common) + desktop-only `dressi_examples_desktop` (vk_viewer,
  desktop_host, glad/glfw) + the six thin executables.
- Verified: ctest green; all six examples run — image_fitting exit 0
  (max err 0.0000), silhouette 60-iter exit 0 (IoU 0.909), pbr_shading
  exit 0, texopt/pbrmat exit 1 only due to shortened-run quality gates,
  pbrenv exit 0. `dressi_examples_core` also compiles for arm64.

### M2b — Android app (`android/`, Gradle + Kotlin + JNI)

- `android/app/src/main/cpp/CMakeLists.txt` `add_subdirectory`s the repo
  root with `DRESSI_BUILD_TESTS=OFF`, `DRESSI_FETCH_DATA=OFF` and builds
  `libdressi_examples.so` (jni_bridge, android_host, device_caps).
- Native layer:
  - `SurfaceState` singleton (mutex-serialized): owns the ANativeWindow,
    stream registry, selected stream; `blit()` = float→RGBA8 nearest-scale
    letterbox via `ANativeWindow_lock`/`unlockAndPost`. surfaceDestroyed
    blocks on the same mutex (Surface contract).
  - `AndroidHost` viewers = streams (all valid; never "close" — Stop
    cancels), so e.g. texture_optimization's three desktop windows become
    three selectable stream buttons.
  - spdlog default logger = `android_sink_mt` (tag `dressi`) +
    `callback_sink_mt` → Kotlin listener (`onLog`).
  - JNI (`NativeBridge`): listExamples (name|needs_geometryShader),
    queryDeviceCaps (JSON via a throwaway vkw instance), startExample
    (one `std::thread` runner at a time, cooperative cancel), requestStop,
    setSurface/selectStream, setListener (onLog/onStreamsChanged/
    onFinished).
- Kotlin (no androidx deps, programmatic UI): MainActivity = caps panel +
  example buttons (silhouette grayed without geometryShader; asset-missing
  examples grayed with reason) ⇄ run panel (SurfaceView + stream bar +
  log tail + Stop/Back), keep-screen-on during runs. `AssetInstaller`
  extracts APK assets to `filesDir/data` (version-marker re-extract).
- Assets: `tools/downsample_exr` (uses new `SaveImageExr` in asset_utils;
  half/ZIP) baked `data/suburban_garden_512.exr` (4096x2048 → 512x256,
  524 KB — the PBS examples immediately downsample to ≤512 anyway).
  Gradle `syncDressiAssets` copies bunny + Avocado + DamagedHelmet + the
  small EXR into APK assets before every build.
- Versions: AGP 8.13.2, Gradle 8.14.3 (wrapper vendored), Kotlin 2.2.20,
  compileSdk/targetSdk 36, minSdk 29, NDK 28.2, SDK CMake 3.31.6,
  `c++_static`.

## Key decisions

- **glslang from source at the lockstep tag** instead of prebuilts or
  VulkanWrapper's clone scripts: matches the repo's FetchContent
  convention and guarantees SPIRV-Tools/Headers compatibility.
- **Software blit viewer** (`ANativeWindow_lock`): the desktop viewer is
  already a CPU-image presenter, so this is the equivalent design with
  zero Vulkan surface/swapchain lifecycle risk. EGL is not involved
  anywhere (Vulkan engine).
- **Streams, not windows**: examples keep creating N viewers; Android
  shows the selected one. `update()` for unselected streams is one mutex +
  compare.
- **Cooperative cancel** at iteration boundaries — normal teardown path,
  no thread kill, GPU state stays consistent.
